# MCA Major Project Report Format – Guidelines

This document outlines the formatting rules for an MCA major project report as per the provided template. Use it with GitHub Copilot to ensure compliance.

## General Layout
- **Paper Size**: A4 (210 mm × 297 mm)
- **Paper Quality**: White bond, ≥95% whiteness, ≥70 gsm
- **Margins**:
  - Top: 1.0 inch
  - Bottom: 1.0 inch
  - Left: 1.5 inches
  - Right: 1.0 inch
- **Line Spacing**: 1.5 lines throughout
- **Font**: Times New Roman
  - Body text: 12 pt
  - Chapter number: 16 pt, bold, sentence case
  - Chapter title: 16 pt, bold, ALL CAPS
  - Section headings: 12 pt, bold, sentence case
  - Subheadings: 12 pt, bold, sentence case
- **Page Numbering**:
  - No number on title page
  - Front matter (declaration to abbreviations): Roman numerals (i, ii, iii...) at left bottom
  - Main body (Chapters onward): Arabic numerals (1, 2, 3...) at left bottom
- **Footer**: Project title on every page after the title page

## Structure & Order of Pages
1. Title page (no page number)
2. Candidate’s declaration (Roman numeral)
3. Certificate (Roman numeral)
4. Acknowledgements (Roman numeral)
5. Abstract (Roman numeral)
6. Table of Contents (Roman numeral)
7. List of Tables (Roman numeral)
8. List of Figures (Roman numeral)
9. List of Abbreviations (optional, Roman numeral)
10. Chapters (Arabic numerals)
11. References
12. Appendices

## Chapters
- Each chapter starts on a fresh page.
- Chapters and sections are numbered with decimals (e.g., 2.1, 2.3.4).
- **Chapter 1 – Introduction**: background, methodology, outline of results/future scope.
- **Chapter 2 – Literature Survey & Problem Outline**: evaluate prior work, gap analysis, problem formulation, objectives, methodology, organization of report.
- **Chapters 3–5**: in-depth details (design, implementation, experiments, results). Include figures, tables, flowcharts, graphs.
- **Penultimate chapter – Results & Discussions**: evaluation of investigation, contributions.
- **Final chapter – Conclusion & Future Scope**: summary, enumerated conclusions, future work.

## Figures and Tables
- **Table caption**: above the table, font size 10, bold (e.g., `Table 1.1: Title`), rest simple.
- **Figure caption**: below the figure, font size 10, bold (e.g., `Figure 1.1: Title`), rest simple.
- Numbering: chapter‑wise (e.g., Figure 5.4 = fourth figure in chapter 5).
- Placement: immediately after first mention. Small tables/figures inline; large ones on separate pages.
- If reproduced from elsewhere, cite source in caption: `(Name of the table [12])`.

## Citations & References
- **In‑text citation**: only last names, chronological order.  
  Example: `Ignizio [3, 4]`, `Badri et al. [1]`.
- **Reference list**: sort by year descending (newest first). Number all references.
- **Format examples**:
  - Journal: `[1] Badri, M.A., ... 1998, "Title", Journal, Vol.110(2), pp.243-260.`
  - Book: `[5] Ignizio, J.P. and Cavalier, T.M., 1994, "Linear Programming", Prentice-Hall, pp.457-505.`
  - Conference: `[2] Chatani, E., et al., 2002, "Title", Proc. of First Int. Conf. on ..., Kyoto, pp.27-32.`
  - Dissertation: `[3] Ignizio, J.P., 1968, "A Method to Achieve...", MS Dissertation, Univ. of Alabama.`

## Equations
- Use Equation Editor or MathType.
- Center alignment, numbered only if referenced (e.g., `(1.1)` flushed right).
- Variables in running text should also use equation tools (inline).
- Do not use bold for variables; bold denotes vectors.

## Appendices
- Numbered with Roman capitals (Appendix A, B, C...).
- Title: font size 16, bold, ALL CAPS.
- Content: lengthy derivations, raw data, code listings, etc.

## Binding & Cover
- Hardbound, maroon cover, lettering embossed in gold.
- Spine: degree, candidate name, year (embossed in gold).
- Two blank white sheets at beginning and end.

## Abbreviations (optional)
- Single page, alphabetical order, font size 12.
- Example:
| Abbreviation | Meaning |
|--------------|---------|
| ANN | Artificial Neural Network |
| NFC | Neuro Fuzzy System |

## Important Reminders
- The abstract is a concise summary of the entire work (1.5 line spacing, paragraph format).
- All paragraphs must start with one tab space.
- Do not use abbreviations without defining them first (unless list is provided).
- Every reference cited in text must appear in the reference list, and vice‑versa.
- Use past tense in the literature survey.
